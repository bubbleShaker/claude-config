# twitterapi MCP の脅威モデル（凍結リスク）とコスト概算

公式 `@kaitoinfra/twitterapi-io-mcp-server`（read 専用）を使うときの
「凍結の恐れ」への不安整理と、料金の概算メモ。
セットアップ手順は [twitterapi-mcp-setup-2026-06-28.md](./twitterapi-mcp-setup-2026-06-28.md) を参照。

---

## 1. 脅威モデル（誰が誰の何を見るか）

通信経路はこうなっている:

```
あなた ──API key── api.twitterapi.io ──(業者の自前アカウント群＆IP)── X(Twitter)
```

- X が見るのは **TwitterAPI.io 側の IP とアカウント**だけ。
  **こちらの IP も X アカウントも X からは一切見えない。**
- こちらの IP を見るのは契約業者 **TwitterAPI.io だけ**。

### 結論1: X アカウント凍結の経路は実質無い

- 公式版は **read 専用**で、`login_user` も `create_tweet` も無い。
  そもそも自分の X アカウントを食わせる口が無い（=設定していない）。
- 凍結リスクを負うのは業者のプール口で、それは **業者が肩代わり**している。
- 「X があなたを IP ブロック／凍結する」は **起こり得ない**（X はこちらの存在を知らない）。

> スクレイピングで凍結が語られるのは、たいてい *自分の X アカウントでログインさせて自動操作する*
> タイプのツール。今回はそれに該当しない。
> 補足: 個人公開の kinhunt 版には login/post ツールがあった。公式版移行(#26)でこの凍結ベクタは消えた。

### 結論2: IP 偽造は不要かつ逆効果

- X はこちらの IP を見ないので、**偽造する相手がいない**（偽造しても X には届かない）。
- 唯一 IP を見るのは契約業者 TwitterAPI.io。それを欺くのは **ToS 違反 → API キー BAN** を招くだけ。
  自分の契約先を回避する行為で、完全に逆効果。やらない。

### 実際に管理すべきリスク（凍結ではない）

| リスク | 対策 |
|---|---|
| 業者 ToS 違反での **API キー BAN** | 常識的な用途・量に留める。回避策(IP偽造等)をしない |
| 過剰リクエストの **レート/課金** | 取得頻度を抑える。取得済み ID をキャッシュして差分取得（#29） |
| 将来 **投稿系を足した時** | 本物アカウントが要り凍結リスクが復活。足さない or 専用捨てアカ前提 |

## 2. コスト概算（2026-06-28 時点の単価）

課金は **リクエスト回数ではなく返却された「件数」ベース**。
1 API コールあたり最低 15 credits（$0.00015）の下限あり。月額固定費なし（従量課金）。
（円換算は **1 USD ≈ ¥155** 想定。為替で変動するので目安。）

### 単価

| 取得対象 | 単価 | 1件あたり |
|---|---|---|
| ツイート | $0.15 / 1,000件 | $0.00015 ≈ ¥0.023 |
| ユーザープロフィール | $0.18 / 1,000件 | $0.00018 ≈ ¥0.028 |
| フォロワー/フォロー中 | $0.01〜 / 1,000件（段階制） | — |

### 行動 → 金額（ミクロ感覚）

| やったこと | 件数 | 金額 |
|---|---|---|
| ツイート1件読む | 1 | ¥0.02（ほぼ無料） |
| 検索で20件返る | 20 | ¥0.5 |
| あるユーザーの直近100件取得 | 100 | ¥2.3 |
| プロフィール1人 | 1 | ¥0.03 |

→ 無料クレジット $0.1（≈ツイート667件分）で試用はまず使い切らない。

### 月の使い方 → 月額（マクロ概算）

| 規模 | 月の取得件数 | 月額(USD) | 月額(円) |
|---|---|---|---|
| ライト（数アカ巡回・1日50件） | 1,500 | $0.23 | ¥35 |
| 中（10アカの新着収集・1日200件） | 6,000 | $0.90 | ¥140 |
| やや重（検索ベース定期収集） | 10,000 | $1.50 | ¥230 |
| 重（大量収集） | 100,000 | $15 | ¥2,300 |
| 超重 | 1,000,000 | $150 | ¥23,000 |

### まとめ

- 個人の定期収集（〜1万件/月）なら **月 ¥200〜300 程度**。
- 1件0.02円なので、ちょっと読む程度なら **実質タダ**。
- 注意は「定期ジョブで同じものを再取得して件数を積む」パターンだけ。
  → ここを **取得済みキャッシュ層（#29）** の差分取得で削れる。
